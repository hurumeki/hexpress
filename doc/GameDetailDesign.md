# **ヘキサ・スライド・パズル：レールパターンとデータ構造詳細定義**

六角形の各辺に 0（真上）から時計回りに 5 までの番号（インデックス）を割り当てて定義します。

### **1\. タイル内レールの基本パターン**

* **直線 (Straight)**: 向かい合う辺を繋ぐ（例: 0番 ↔ 3番）。  
* **浅いカーブ (Shallow Curve)**: 隣接する辺を繋ぐ（例: 0番 ↔ 1番）。  
* **深いカーブ (Deep Curve)**: 一つ飛ばしの辺を繋ぐ（例: 0番 ↔ 2番）。  
* **交差点 (Crossing)**:  
  * 独立交差：レールが2本あり、互いに干渉せずに通り抜ける。  
  * 合流・分岐：3方向以上が1点で繋がり、入った方向によって次の駒を押し出す先が決まる。  
* **一方通行 (One-Way)**: Aから入るとBに出られるが、Bから入ることはできない。  
* **デッドエンド (Dead End)**: 駒が入るとそれ以上進めない。

### **2\. 駒と色の定義（26種類＋中立）**

* **クリア対象の駒（25種）**:  
  * **色 (5色)**: 朱（木）、藍（石）、若草、黄金、墨。  
  * **模様 (5種)**: 輪（円）、角（四角）、菱（ひし形）、三（横線）、点（ドット）。  
* **中立の駒（1種）**:  
  * **色**: 生成色（ベージュ/白木）など。  
  * **模様**: なし。クリア条件の判定には含まれない。

### **3\. ステージデータの構成案（AI指示用）**

各ステージは以下のプロパティを持つオブジェクトとして定義します。

* layout: 盤面の形状（ヘキサ座標リスト）。  
* tile\_configs: 各座標のタイルの設定。  
  * rails: レールの接続リスト（例: \[{from: 0, to: 3, flow: 'both' | 'one-way'}\]）。  
  * target\_id: そのタイルに置くべき「正解の駒」のID（空タイルの場合はnull）。  
* initial\_placement: 初期状態の駒の配置（駒がないタイルはnull）。  
* entry\_points: 盤面外から駒を差し込める辺の座標とインデックスのペア。  
* initial\_hands: 最初にプレイヤーが持っている駒のリスト（配列）。

### **4\. 操作ロジック（玉突きと手札管理）**

1. **駒の選択**: プレイヤーは hands（手札）の中から次に差し込む駒を1つ選択する。  
2. **駒の投入**: 選択した駒を entry\_point から差し込む。  
3. **移動と玉突き**:  
   * 投入された駒がレールの接続先に進む。  
   * 移動先にすでに駒がある場合、その駒がさらにレールの先（隣接タイル）へ押し出される。  
   * **空きタイルの挙動**: 移動先に駒がない場合、駒はそのタイルに留まり、玉突きは終了する。  
4. **手札の更新**:  
   * レールの終端が「盤面外」に達し、駒が押し出された場合、その駒がプレイヤーの hands に追加される。  
   * 使用した駒は hands から削除される。

### **5\. クリア条件**

* target\_id が設定されているすべてのタイルにおいて、載っている駒の「色と模様」がタイルの指定と一致した瞬間にクリアとなる。
